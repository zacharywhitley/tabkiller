# Task: Capture layer consolidation

## Description

Retire the older `src/tracking/*` layer, hook `webNavigation.onCommitted`
so `transitionType` is populated on every navigation, and unify the
`tabs.onActivated` / `windows.onFocusChanged` streams into a single
focus-transition stream. After this task, the capture surface emits every
signal the graph model needs; T4 will consume it.

## Acceptance Criteria

- [ ] `src/tracking/*` is deleted. Any unique session-detection heuristics
      that lived only in `src/tracking/SessionTracker.ts` are migrated
      into `src/session/detection/*` (which already has the "advanced"
      engine).
- [ ] `webNavigation.onCommitted` is hooked in the background service
      worker. `webNavigation.transitionType` is captured on every
      committed navigation and included in the emitted event.
- [ ] `webNavigation` is added to `manifest.json` permissions (all
      four browser targets).
- [ ] `tabs.onActivated` and `windows.onFocusChanged` are reconciled
      into a single focus-transition emitter. Emitter contract:
      exactly one "focus is now on visit V" or "focus is now on
      nothing" event per underlying transition.
- [ ] Opener chain is captured at tab creation as an outbox event
      that names the parent visit (not the parent tab). The parent
      visit id must be resolvable to a durable node id, not a
      transient browser tab id.
- [ ] Existing session-detection tests still pass. New tests cover the
      focus-reconciliation logic and the webNavigation event shape.

## Technical Details

- **Migration path:** grep `src/tracking/` imports; every consumer
  should already have an equivalent in `src/session/tracking/*` â€”
  swap the imports, then delete the files.
- **`webNavigation.onCommitted` filter:** apply
  `{url: [{schemes: ['http', 'https']}]}` so `chrome://` / `about:`
  navigations don't spam the outbox.
- **Focus reconciliation:** the browser fires *both* `tabs.onActivated`
  and `windows.onFocusChanged` in some transitions; the emitter must
  dedup so downstream code never sees a duplicate "gained focus" event
  for the same visit.
- **Parent visit at tab creation:** at `tabs.onCreated`, resolve
  `openerTabId` to the currently-active visit in that tab and record
  its id in the emitted event. If the opener tab has no known active
  visit, emit the event without an opener â€” do not block.
- **MV3 service worker termination** can drop in-flight focus events.
  Mitigation is downstream (T4 reconciles on visit close), but this
  task should not add its own state that depends on the worker
  staying alive.
- **Files affected:**
  - Deleted: `src/tracking/EventTracker.ts`, `src/tracking/TabTracker.ts`,
    `src/tracking/NavigationTracker.ts`, `src/tracking/SessionTracker.ts`,
    `src/tracking/WindowTracker.ts` (and directory)
  - Modified: `src/background/service-worker.ts` (webNavigation hook,
    focus emitter wiring), `src/session/tracking/TabLifecycleTracker.ts`
    (add webNavigation-source events), `src/session/detection/*`
    (absorb any unique heuristics from the deleted SessionTracker),
    `manifest.json` (permission), per-browser manifest templates if
    they exist under `src/manifest/*`
  - New: `src/session/tracking/FocusEmitter.ts`
