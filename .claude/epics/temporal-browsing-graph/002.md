# Task: URL normalization

## Description

Pure function that canonicalizes a URL to define `Page` identity. This is
where silent corruption hides — wrong normalization overcounts or
undercounts Page nodes and breaks every downstream query — so it is
isolated as its own tested module.

## Acceptance Criteria

- [ ] `src/utils/url.ts` exports `normalizeUrl(raw: string): string`.
- [ ] Behavior:
  - Scheme and host lowercased.
  - Strip tracking params from a hardcoded allowlist:
    `utm_source`, `utm_medium`, `utm_campaign`, `utm_term`,
    `utm_content`, `fbclid`, `gclid`, `mc_cid`, `mc_eid`, `_ga`, `ref`,
    `ref_src`, `igshid`, `mkt_tok`, `yclid`, `vero_id`, `_hsenc`,
    `_hsmi`, `hsCtaTracking`.
  - Preserve every other query param, including empty ones.
  - Sort query params by key for stable identity.
  - Strip fragment by default.
  - Per-domain fragment allowlist (SPAs that use fragments as routes)
    — start with an empty allowlist; the mechanism must exist and be
    testable.
  - Remove trailing slash from path, unless path is exactly `/`.
  - Percent-encoding normalized to lowercase hex.
- [ ] Malformed URLs return the raw input unchanged and log a warning
      (via existing logger) — do not throw.
- [ ] `src/utils/__tests__/url.test.ts` has at least 20 cases covering:
      every tracking param stripped, unknown params preserved, sorting,
      fragment strip, per-domain fragment kept, trailing slash, case
      folding, IDN hosts, malformed input.

## Technical Details

- **Pure function.** No side effects, no dependencies beyond `URL`.
- **Use `URL` constructor** for parsing; catch `TypeError` for malformed.
- **Per-domain fragment allowlist** as `Map<string, boolean>` exposed
  via a setter for future settings integration; empty by default.
- **Log unknown query params on ingest** (not in this task — in T4) so
  we can extend the allowlist based on real captured data.
- **Files affected:**
  - New: `src/utils/url.ts`, `src/utils/__tests__/url.test.ts`
