# Task: Cutover + delete pass

## Description

Delete everything the new graph store replaces. Remove the additive
dual-write path added in T4. Verify the bundle-size delta and confirm no
new runtime dependencies were added.

## Acceptance Criteria

- [ ] `level-browserify` removed from `package.json` and
      `package-lock.json`; any transitive Level dependencies gone.
- [ ] `src/database/connection.ts`'s LevelGraph paths deleted; if the
      file has no remaining purpose, delete it entirely.
- [ ] `src/database/schema.ts` reduced to only the edge types actually
      used by the graph model. Unused declarations (`SYNCED_FROM`,
      `ACCESSED_BY`, `CONTAINS_TAB`, and any others confirmed unused
      after this epic) removed.
- [ ] `src/database/repositories.ts` reviewed; any repository that only
      exists to serve the deleted LevelGraph path is removed.
- [ ] The additive dual-write path added in T4 is removed — writes go
      only to the new `GraphStore`. `LocalEventStore`'s durable-storage
      role is fully replaced by its outbox role; nothing reads from it
      as a query source anymore.
- [ ] Bundle-size delta ≤ 50 KB relative to the pre-epic baseline.
      Measured, not asserted.
- [ ] All existing tests still pass.
- [ ] `npm ls` shows no `level-*` packages.
- [ ] Extension loads and works end-to-end after the cutover across
      Chrome and Firefox targets.

## Technical Details

- **Order of deletion matters:** first delete callers, then callees.
  Compilation errors are the guide. Do not use `//@ts-ignore` or
  `any` to paper over — if a delete breaks a caller, the caller was
  wrong to depend on the deleted thing.
- **Baseline for bundle-size delta:** compare against the last
  production build before this epic (whichever commit `main` was on
  at PRD creation). Use `webpack-bundle-analyzer` or the existing
  build scripts.
- **`SessionStorageEngine` remains** — it hosts the new `graph_nodes`
  / `graph_edges` object stores. Only its old session/tab/navigation
  stores are candidates for removal, and only if T5's real-day
  validation confirmed the graph model fully replaces them for query
  purposes. If the UI still consumes the old stores, defer their
  deletion to the follow-up UI epic; only remove the *graph-side*
  dead code here.
- **Files affected:**
  - Deleted: `src/database/connection.ts` (or LevelGraph paths in it),
    unused entries in `src/database/schema.ts`,
    unused files in `src/database/repositories.ts`
  - Modified: `package.json`, `package-lock.json`,
    `src/database/graph/ingest.ts` (drop dual-write),
    background service worker startup (drop LevelGraph init)
