# Task: IndexedDB GraphStore

## Description

Port the schema types and store API from `spike/temporal-graph/` into
`src/database/graph/`, backed by real IndexedDB via the existing
`SessionStorageEngine` (schema version bump). Every read and write goes
through this store; no code should ever open IndexedDB for graph data
outside `GraphStore`.

## Acceptance Criteria

- [ ] `src/database/graph/types.ts` exists as a straight port of the spike's
      `types.ts` (identical field names and shapes).
- [ ] `src/database/graph/store.ts` exposes the same public API as the
      spike store (`putNode`, `putEdge`, `getNode`, `nodesOfType`,
      `outEdges`, `inEdges`, `outPoint`, `outInterval`, `inPoint`,
      `inInterval`, `edgesOfType`, `stats`), backed by IndexedDB.
- [ ] `SessionStorageEngine`'s IndexedDB schema is bumped to add two
      object stores: `graph_nodes` and `graph_edges`.
- [ ] Compound indexes are created per the PRD:
      - `graph_nodes`: `(type, at_time)`, plus type-specific singles
        for `Page.normalized_url`, `Domain.hostname`, `Tag.slug`,
        `Tab.browser_tab_id`.
      - `graph_edges`: `(from_id, type)`, `(to_id, type)`, `(type, at_time)`.
- [ ] Every write is atomic in a single IndexedDB transaction (node +
      any outgoing edges emitted in the same event go together).
- [ ] Store has a jest test covering: put/get for each node type,
      point-edge round-trip, interval-edge round-trip, retrieval via
      each named index.
- [ ] All timestamps stored as integer ms-since-epoch — no `Date`
      objects, verified by tests reading raw records.

## Technical Details

- **Reuse:** `SessionStorageEngine` opens the IndexedDB instance and
  handles versioning. Extend its `onupgradeneeded` handler; do not
  create a second database.
- **API parity:** the store class's method signatures must match the
  spike so the queries in T5 port with minimal changes.
- **Discriminator:** point/interval edges live in the same object store
  and are distinguished by a `kind: 'point' | 'interval'` field.
- **Transactions:** IndexedDB transactions are per-object-store or
  cross-store; graph writes require a `readwrite` transaction spanning
  both `graph_nodes` and `graph_edges`.
- **Files affected:**
  - New: `src/database/graph/types.ts`, `src/database/graph/store.ts`,
    `src/database/graph/__tests__/store.test.ts`
  - Modified: `src/session/storage/SessionStorageEngine.ts` (version
    bump + new stores), `src/session/storage/schema.ts` (index
    declarations)
