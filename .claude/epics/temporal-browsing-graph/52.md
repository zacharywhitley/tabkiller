# Task: Ingest pipeline

## Description

Drain events from `LocalEventStore` (repurposed as a write-ahead outbox)
and transform them into `nodes` and `edges` writes on the `GraphStore`.
Runs in additive mode: existing storage paths continue to work, so the
old system remains functional until T5 validates and T6 cuts over.

## Acceptance Criteria

- [ ] `src/database/graph/ingest.ts` exposes `drainOutbox()` that:
      reads pending events from `LocalEventStore`, transforms each into
      graph writes, commits them atomically per event, and marks the
      event drained on success. Never loses events on failure.
- [ ] Event â†’ graph transformer covers every event type produced by T3:
      tab created / updated / removed, `webNavigation.onCommitted`,
      focus transition, session start/end, tag applied, tag removed.
- [ ] Every transform emits its node + all outgoing edges inside a
      single IndexedDB `readwrite` transaction spanning `graph_nodes`
      and `graph_edges` â€” if any write fails, none are applied.
- [ ] `chrome.alarms` schedules the drain every 30s (constant, easy to
      tune); an immediate drain is also triggered by the emitter after
      each event so hot writes don't wait for the alarm.
- [ ] Unknown query params encountered during URL normalization are
      logged (structured log line, deduplicated by param name) for
      later allowlist extension.
- [ ] Focus intervals on `Visit` are reconciled on visit close: any
      trailing interval with `end === null` is closed to the visit's
      `ended_at`.
- [ ] Old storage paths (SessionStorageEngine's per-session records,
      LocalEventStore's raw batches) continue to work â€” this task is
      additive, not a cutover.
- [ ] Ingest is idempotent: running the same batch twice produces the
      same graph state (transformer uses event ids as node/edge ids
      where possible; dedup writes when they aren't).
- [ ] Tests: one per event type; one for atomicity (throw mid-transform,
      verify no partial writes); one for the alarm-driven drain.

## Technical Details

- **Outbox drain semantics:** `LocalEventStore` already batches; extend
  it (do not replace) with a `markDrained(eventIds: string[])` method.
  Failed drains do not mark; next tick retries.
- **Event â†’ graph mapping:**
  - `tab_created` â†’ `Tab` node + `in_window` interval edge; if opener
    is known, emit `opened_from` point edge from the new tab's first
    Visit (created lazily on first navigation) â€” until then, buffer.
    Note: this is the one non-trivial ordering â€” an `opened_from` edge
    requires the child Visit to exist, which only happens on the first
    `webNavigation.onCommitted` in that tab. Buffer the opener info
    keyed by `browser_tab_id` and consume on first navigation.
  - `webNavigation.onCommitted` â†’ `Visit` node (new) +
    `of_page` interval edge + `in_tab` interval edge + `in_session`
    interval edge + `navigated_from` point edge to the previous Visit
    in the same tab (if any) + close prior Visit's `ended_at`. If a
    search results URL is detected, also emit `SearchQuery` node
    and, on the *next* Visit, an `arrived_via` point edge.
  - `focus_transition` â†’ append/close an entry in the current Visit's
    `focus_intervals` (materialized on write; the source-of-truth
    focus event log is not retained).
  - `session_started`/`session_ended` â†’ `Session` node create /
    `ended_at` close; adjust affected Visits' `in_session` edges by
    clipping intervals (spike showed v5-style multi-session visits
    need two edges).
  - `tag_applied` â†’ `Tag` node (create-if-not-exists by slug) +
    `tagged_with` point edge from Session.
- **Idempotency:** derive node/edge ids deterministically from event
  ids where possible (`visit_<eventId>`, `edge_<eventId>_<kind>`),
  so replays don't create duplicates.
- **Search-engine detection:** small regex list (google.com/search,
  bing.com/search, duckduckgo.com/, youtube.com/results). Extract
  `q` param as query text. Extensible list; not comprehensive at v1.
- **Files affected:**
  - New: `src/database/graph/ingest.ts`,
    `src/database/graph/transformers.ts`,
    `src/database/graph/__tests__/ingest.test.ts`
  - Modified: `src/storage/LocalEventStore.ts` (add drain helpers),
    `src/background/service-worker.ts` (alarm registration, immediate
    drain hook)
