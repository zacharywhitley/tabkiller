# Task: Query API + real-day integration

## Description

Port the six query primitives from `spike/temporal-graph/queries.ts` into
`src/database/graph/queries.ts`, backed by the real `GraphStore`. Then
capture a real day of the developer's own browsing, run every query
against it, and verify results by hand. This is the epic's acceptance
test.

## Acceptance Criteria

- [ ] `src/database/graph/queries.ts` exports six primitives with the
      same signatures as the spike (adjusting the Q4 split from the PRD):
      - `pagesOpenedFromDomain(hostname, from, to)`
      - `visitFocusedAt(t)`
      - `visitsInTagPredatingTag(tagSlug)`
      - `causalPredecessors(visitId, windowMs)` (renamed from `visitsBefore`
        per the PRD's Q4 decision)
      - `visitsOnScreenBetween(tFrom, tTo)` (new — the temporal-window
        primitive the spike showed was needed)
      - `tabTreeForTag(tagSlug)`
- [ ] Each query runs against the ported spike fixture (also moved into
      `src/database/graph/__tests__/fixture.ts`) and returns identical
      output to the spike's run.
- [ ] `visitsOnScreenBetween` covers the case Q4 in the spike revealed:
      the immediately-preceding visit whose `[at_time, ended_at]`
      overlaps the window is returned, even if `at_time` is far outside
      the window.
- [ ] **Integration test (manual):** the developer installs the built
      extension, uses their browser normally for one day (target: 100+
      navigations, multiple sessions, at least one retroactive tag).
      Then runs each query and verifies:
      - Q1 returns pages actually opened from a chosen source domain
      - Q2 answers "what was I looking at at 3:47pm?" correctly for a
        remembered instant
      - Q3 returns pre-tag visits after a retroactive tag application
      - Q4a walks a real causal chain across intra-tab navigation
      - Q4b returns everything visible during a chosen 10-minute window
      - Q5 reconstructs a tab tree recognizable to the developer
- [ ] Findings from the real-day run — surprises, wrong answers,
      unexpected data shapes — are documented in
      `.claude/epics/temporal-browsing-graph/real-day-findings.md`.
- [ ] Query-level unit tests (fixture-based) added for each primitive.

## Technical Details

- **Cursor walks over IDB indexes** replace the spike's Map scans.
  Every primitive should use the indexes named in T1's schema; no
  full-table scans in the query path.
- **`visitsOnScreenBetween` implementation:** cursor over the `Visit`
  `(type, at_time)` index starting from `at_time <= tFrom`, walking
  backward until `ended_at` no longer overlaps the window; forward
  cursor from `tFrom` for visits that started inside the window.
- **Real-day capture protocol:** developer uses the extension in a
  regular profile for one day; at end of day, opens the extension's
  debug console (temporary view added if needed) and runs each query
  with realistic parameters. Not automated — the whole point is to
  see whether the data *feels right*.
- **Debug console for real-day validation** — a minimal options-page
  panel that lets the developer paste query args and see results.
  Not shipped to users; can be scoped behind a debug flag or removed
  after this task.
- **Files affected:**
  - New: `src/database/graph/queries.ts`,
    `src/database/graph/__tests__/queries.test.ts`,
    `src/database/graph/__tests__/fixture.ts`,
    `src/options/debug/GraphQueryPanel.tsx` (temporary),
    `.claude/epics/temporal-browsing-graph/real-day-findings.md`
